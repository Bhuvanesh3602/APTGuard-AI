HOW TO READ THIS FILE
---------------------
Each tab below shows:
  • Label        — the text shown in the left sidebar
  • Route        — the URL it opens (prefix with http://localhost:3000)
  • Renders      — the React component that draws the page
  • Does         — what it actually shows/does when you click it today
  • Data source  — where the numbers come from RIGHT NOW:
        [LIVE]   = calls a real backend API / WebSocket; numbers are real
        [STATIC] = hardcoded demo data inside the React component
                   (representative of a real attack; not wired to a backend yet)

BIG PICTURE (one line)
----------------------
  • The ORIGINAL AiSOC platform tabs (Dashboard, Alerts, Queue, Cases,
    Playbooks, Copilot, Attack Graph, Threat Hunt, MITRE Coverage, Connectors)
    are [LIVE] — they fetch from the api/agents/actions services.
  • The NEW India-CNI tabs (CNI Dashboard, UEBA, Correlation, APT Intelligence,
    Vulnerability Prioritisation, EoL Risk Map, OT Topology, Digital Twin,
    CERT-In Compliance) are [STATIC] — they render curated demo data that mirrors
    the real backend behaviour, but the UI is not yet calling the live CNI
    services. The live CNI service APIs DO exist (see "BACKEND READY" at bottom).

The sidebar is organised into 7 sections that map to the 5 hackathon pillars.

 SECTION: COMMAND CENTRE
================================================================================

1) CNI Threat Dashboard
   Route:   /cni
   Renders: components/cni/CNIDashboard.tsx
   Does:    The India-first landing screen. Shows a live IST clock, 4 KPI cards
            (Active CNI incidents, CERT-In reports overdue, EoL systems at risk,
            APT groups active), a per-sector incident summary (Healthcare /
            Education / Power Grid / Govt IT), and four panels: CERT-In 6-hour
            reporting status, India APT attribution (APT36/SideCopy/Volt Typhoon/
            Lazarus with confidence %), EoL CVSS amplification, and an OT/ICS
            incident feed. Ends with an APT kill-chain stage predictor.
   Data:    [STATIC] — all values are hardcoded demo data (the clock is the only
            live element). File comment: "Static demo data representing the CNI
            threat landscape".

2) SOC Metrics
   Route:   /dashboard
   Renders: components/dashboard/DashboardView.tsx
   Does:    The classic SOC operations dashboard — funnel KPIs (events → alerts →
            cases), pipeline health, live event feed, efficiency report, MTTD/MTTR.
   Data:    [LIVE] — fetches /api/v1/metrics/dashboard and related endpoints from
            the api service. (If dev-auth/env is misconfigured it falls back to
            baseline mock data and retries — see the local-dev env note.)

 BACKEND READY (live CNI service APIs the STATIC tabs can be wired to)
================================================================================
These services are implemented and run real logic. The STATIC tabs above show
demo data today, but each has a live API it could fetch from:

  • Digital Twin Simulator  ->  digital-twin service (http://localhost:8011)
        POST /api/v1/twin/simulate            (scenario attack path)
        POST /api/v1/twin/what-if             (control risk-reduction)
        GET  /api/v1/twin/graph/status        (Neo4j vs static fallback)
        GET  /api/v1/twin/graph/assets        (live asset inventory)
        GET  /api/v1/twin/graph/reachable/{asset_id}
        GET  /api/v1/twin/graph/attack-path?from_asset=&to_asset=
        GET  /api/v1/twin/graph/blast-radius/{asset_id}
        (now backed by a real Neo4j graph; static BFS fallback if DB is down)

  • EoL Risk Map / Vulnerability Prioritisation -> eol-prioritiser (http://localhost:8010)
        POST /api/v1/eol/scan                 (EoL × CVSS amplification, live CVE)
        POST /api/v1/eol/scan/bulk            (risk-ranked remediation queue)
        GET  /api/v1/eol/cve/{cve_id}         (live NVD CVSS + CISA KEV status)
        GET  /api/v1/eol/kev/status           (CISA Known-Exploited catalog state)
        (now pulls live CVSS from NVD and known-exploited flag from CISA KEV)

  • India APT Intelligence / CERT-In Compliance -> agents (http://localhost:8001)
        POST /api/v1/cni/apt/attribution      (India APT attribution)
        POST /api/v1/cni/apt/prediction       (next-stage ATT&CK prediction)
        POST /api/v1/cni/certin/report        (CERT-In 6-hour report generation)
        POST /api/v1/cni/full-assessment      (all 5 CNI agents; ?execute=true
                                               also dispatches SOAR actions)
        POST /api/v1/cni/actions/dispatch     (ProposedAction -> actions executor)
        GET  /api/v1/cni/certin/rag/search?q= (live CERT-In advisory RAG / Qdrant)
        GET  /api/v1/cni/certin/rag/status    (is the RAG collection seeded?)

  • Anomaly Detection (UEBA)  ->  ueba service (http://localhost:8007)

NOTE: To make the CNI tabs LIVE, point each component's data load at the
endpoint above (replace the hardcoded arrays with a fetch). All four backend
integrations degrade gracefully, so the tabs will still render if a backend
or its datastore (Qdrant/Neo4j/NVD/KEV) is unavailable.
